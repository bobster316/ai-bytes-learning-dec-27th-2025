The scale is almost absurd

The human brain contains roughly 86 billion neurons. A GPT-4-scale model has roughly 1.8 trillion parameters — each one a number, adjusted billions of times during training. Parameters are not neurons — they are the connection weights between neurons. The number of parameters determines how much the network can memorise and generalise.

How it works

The four parts of every neural network

Every neural network — from a simple classifier to a frontier language model — shares the same fundamental structure. Understanding these four components explains how any AI learns from data.

Input Layer — Part 1 of 4

Data enters as numbers

Every input — a pixel, a word, a temperature reading — is converted to a number before a neural network can process it. There are no exceptions. Images become grids of pixel values. Words become vectors of hundreds of floating-point numbers. This numerical translation is the foundation that makes the maths possible.

This translation step is not trivial. Good representations capture meaningful structure — similar words end up with similar vectors, similar images end up with similar pixel patterns. The quality of this encoding limits everything downstream.

"The word 'cat' might be represented as a vector of 768 numbers — each capturing a different semantic dimension."
Hidden Layers — Part 2 of 4

Patterns emerge in the middle

Hidden layers are where the real work happens. Each neuron takes a weighted sum of its inputs, applies a non-linear function (called an activation function), and passes the result forward. Early layers detect simple patterns — edges, tones, syllables. Deeper layers combine these into complex concepts — faces, sentiment, meaning.

The depth of a network — how many hidden layers it has — determines how abstract its representations can become. This is why deep neural networks are so powerful: each layer builds on the one before it.

"In a vision model, layer 3 might detect 'curved edge facing right'. Layer 12 might detect 'ear of a cat'."
Output Layer — Part 3 of 4

A prediction is made

The output layer produces the network's answer. For classification, this is a probability distribution — the network assigns a likelihood to each possible class. For language models, it's a probability over every word in the vocabulary. The highest probability wins and becomes the prediction.

The output layer design is task-specific. Classification tasks use softmax to produce probabilities. Regression tasks use a linear output. Language models use a vocabulary-sized softmax that can output tens of thousands of possible next tokens.

"Output: [cat: 0.91, dog: 0.07, bird: 0.02] — the network is 91% confident this image shows a cat."
Loss Function — Part 4 of 4

The network measures its own mistake

After each prediction, the loss function calculates how wrong the answer was. A perfect prediction gives a loss of zero. A catastrophically wrong prediction gives a high loss. This single number — the loss — is the signal that drives all learning. Without it, the network has no way to improve.

Common loss functions include cross-entropy (for classification) and mean squared error (for regression). The choice of loss function shapes what the network considers "good" — choose the wrong one and the network optimises for the wrong thing.

"Predicted: cat (0.91). Actual: dog. Loss = 2.4. The network was confident and completely wrong."
The training loop

Two passes. One learning loop.

Every training step involves exactly two passes through the network. The forward pass makes a prediction. The backward pass fixes the mistake. Repeat this millions of times and the network learns.

What makes them powerful

What makes neural networks powerful?

∞
Universal approximators
Given enough layers and data, a neural network can approximate any mathematical function. This is not a metaphor — it is a proven theorem (the Universal Approximation Theorem). It means the architecture itself has no ceiling.
⚠
Hallucination risk
Neural networks do not know when they don't know. They always produce an output, even when the input is completely outside their training distribution. This is why human verification matters for high-stakes decisions.
⇄
Transfer learning
A network trained on 1 billion images can be fine-tuned on 500 medical scans and become a world-class diagnostic tool. The general knowledge transfers. Only the final specialisation needs new data.
↘
Gradient descent
Every weight update follows the direction that reduces loss most steeply. Like a ball rolling downhill, the network finds lower and lower loss valleys through millions of tiny steps.
✦
Emergent capabilities
Scale a neural network enough and unexpected abilities appear — translation, code generation, logical reasoning — that nobody explicitly programmed. These are not designed; they emerge.

The network never resets — it adjusts incrementally. The loss function measures the error, backpropagation calculates how much each weight contributed to that error, and gradient descent nudges every weight in the direction that would have produced a better answer. This happens millions of times per training run.

Explain backpropagation in plain language

A colleague asks: "How does a neural network actually learn from its mistakes?" Use the four parts below to build a clear, jargon-free explanation.

Weak explanation to improve
"The network uses backpropagation to update the weights."
Model answer

Think of it like a chef tasting a dish after it's been served and realising it's too salty. Backpropagation works backwards through every decision that led to that mistake — how much salt each cook added — and tells each one to adjust slightly next time. In a neural network: after a wrong prediction, the loss score flows backwards through every layer, and each weight gets a small nudge in the direction that would have made the answer slightly more correct. Over millions of iterations, these tiny nudges compound into genuine knowledge. In practice: if a model incorrectly classifies a cat as a dog, backpropagation identifies which neuron connections were most responsible and reduces their influence on future cat-vs-dog decisions.

Key concepts in depth

Activation Functions

ReLU (Rectified Linear Unit) is the most widely used activation function. It outputs zero for negative inputs and passes positive inputs unchanged. This simple rule creates the non-linearity that lets deep networks learn complex patterns. Without it, stacking layers would be mathematically equivalent to having a single layer.

Non-linearity is the key. A network with only linear operations — no matter how deep — collapses to a single linear function. Activation functions break this collapse, allowing each layer to learn genuinely different representations.

def relu(x):
    return max(0, x)

# Example:
relu(-3.5)  →  0      # negative: blocked
relu(0)     →  0      # zero: blocked
relu(4.2)   →  4.2    # positive: passes through
Learning Rate

The learning rate controls how large each weight update step is. Too high: the network overshoots the optimal weights and diverges. Too low: training takes impossibly long and often gets stuck. Most modern training uses adaptive learning rates (Adam optimiser) that adjust automatically based on recent gradient history.

Finding the right learning rate is one of the most important hyperparameter choices in training. Too small and the network never converges. Too large and the loss bounces around unpredictably. Adaptive methods like Adam and AdamW largely solve this in practice.

# Learning rate examples:
lr = 0.1      # too high: diverges, loss explodes
lr = 0.000001 # too low: glacially slow, gets stuck
lr = 0.001    # commonly used default (Adam)

# Adaptive: Adam optimiser adjusts lr per-weight
optimizer = Adam(lr=0.001)  # standard starting point
Overfitting

A network that memorises training data instead of learning general patterns. It performs perfectly on training data and fails on new inputs. Signs: training loss falls while validation loss rises. This is the most common failure mode in machine learning.

The fix is regularisation. Dropout randomly disables a fraction of neurons during training, forcing the network to learn redundant representations. L2 regularisation penalises large weights. More training data is usually the most effective cure of all.

# Overfitting example:
training_accuracy   = 99.8%   # memorised the data
validation_accuracy = 61.2%   # generalises badly

# Fix: add dropout
model.add(Dropout(0.3))   # randomly zero 30% of neurons

# After fix:
training_accuracy   = 94.1%   # slightly lower
validation_accuracy = 91.7%   # much better generalisation
Transfer Learning

Take a network already trained on millions of examples, freeze most of its layers (preserving the general knowledge), and only retrain the final few layers on your specific task. This reduces the data and compute required by orders of magnitude.

Transfer learning is why most real-world AI applications are affordable. Fine-tuning a pre-trained model costs a fraction of training from scratch. A small team with a few hundred labelled examples can reach state-of-the-art performance on a specialised task.
