01 / INPUT LAYER

Raw data enters here

The input layer receives the raw numbers that represent your data. For an image, each pixel becomes one input neuron. For text, each token. These neurons don't calculate anything — they simply pass values forward into the network.

Passive receivers
02 / HIDDEN LAYERS

Where understanding forms

Hidden layers do the actual work. Each neuron multiplies its inputs by learned weights, sums them, then passes the result through an activation function. Early layers detect edges. Middle layers detect shapes. Deep layers recognise full concepts like "cat" or "fraud".

Active computation
03 / SECOND HIDDEN LAYER

Higher-level abstraction

Each additional hidden layer learns increasingly abstract representations. The first hidden layer of an image model might detect edges. The second detects textures. The third detects object parts. This hierarchical feature learning is why depth matters.

Abstraction hierarchy
04 / OUTPUT LAYER

The decision

The output layer produces the final answer. In a classifier, each output neuron represents one class. The neuron with the highest activation wins. In GPT, the output is a probability distribution over every word in the vocabulary — choosing the most likely next token.

Final prediction
Comparison

Biology vs Silicon

🧠
Biological Neuron
Your brain has 86 billion neurons, each forming up to 10,000 synaptic connections. Signals travel via electrochemical pulses at around 100m/s. Learning happens through synaptic strengthening — use a connection repeatedly, and it grows stronger. This is Hebb's Rule: neurons that fire together, wire together.
Neural networks are a simplified mathematical abstraction of exactly this mechanism.
86B neurons · 100T synapses
⚡
Artificial Neuron
Takes a list of numbers (inputs), multiplies each by a weight, adds a bias term, passes through an activation function. The entire computation is: output = f(Σwᵢxᵢ + b). That's it.
y = f(Σwx + b)
🔗
Weights = Synapses
Where biology uses synapse strength, AI uses a floating point number called a weight. Positive weights excite. Negative weights inhibit. Training adjusts all weights using backpropagation.
Learned from data
📈
Training Accuracy Over Time
A network starts with random weights — essentially guessing. With each batch of data and weight update, accuracy climbs. The chart shows a real training run — random → expert over thousands of steps.
🚀
Speed Difference
A biological neuron fires ~200 times/second. An A100 GPU performs 312 trillion FP16 operations per second. This raw computational speed enables what biology cannot do in real time.
10⁹× faster

What They Share
Both learn from experience. Both form hierarchical representations. Both generalise from specific examples to new situations. The mathematical inspiration from neuroscience runs deep.

Activation Functions

Choosing the right switch

Every neuron needs a non-linear activation. Without one, the whole network — regardless of depth — collapses into a single linear equation.

Rectified Linear Unit

The workhorse of modern deep learning. Brutally simple: if the input is negative, output zero. If positive, pass it through unchanged. This rule alone solved the vanishing gradient problem that stopped neural networks from getting deep.

f(x) = max(0, x)
StrengthsComputationally cheap · No vanishing gradients · Sparse activations improve efficiency
WeaknessDying ReLU — if weights go very negative, neurons permanently stop firing

Sigmoid

The original activation function — squashes any value between 0 and 1. Perfect for binary classification output layers. Problematic in deep networks because gradients shrink exponentially as they travel backwards through layers.

f(x) = 1 / (1 + e⁻ˣ)
StrengthsSmooth gradient · Output interpretable as probability
WeaknessVanishing gradients · Slow convergence in deep nets · Not zero-centred

Tanh

Hyperbolic tangent — squashes to -1 to +1. Zero-centred unlike sigmoid, which means gradients don't push all weights in the same direction simultaneously. Often outperforms sigmoid in hidden layers as a result.

f(x) = (eˣ - e⁻ˣ) / (eˣ + e⁻ˣ)
StrengthsZero-centred · Stronger gradients than sigmoid
WeaknessStill saturates at extremes — vanishing gradients persist

GELU

Gaussian Error Linear Unit — the activation function used in GPT, BERT, and most modern transformers. Smoother than ReLU: rather than a hard zero cutoff, it smoothly weights inputs by their probability under a Gaussian. Current state-of-the-art for large language models.

f(x) ≈ 0.5x · (1 + tanh(√(2/π)(x + 0.044715x³)))
StrengthsSmooth · No dying neurons · Best empirical results on transformers
WeaknessMore expensive to compute than ReLU
