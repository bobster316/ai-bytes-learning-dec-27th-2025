The Rhythm Model

Every lesson is a scored piece. Five beats.

The Conductor maps every lesson onto a five-beat arc. Block types are assigned emotional weights. The engine sequences them so learners are never in the same emotional state for more than two consecutive blocks.

The Conductor does not force this sequence rigidly. It uses it as a scoring guide — ensuring no lesson is all tension, no lesson is all reward. A clinical-archetype course may spend more time in Calm and Building. A provocateur-archetype course may linger in Tension. But every lesson must pass through all five beats.

Anti-Chaos Rules

The orchestra needs a score, not a jam session.

Constraints are what make the variation feel designed, not random. These rules are enforced by the Conductor at generation time — before blocks are written to the database.

🚫
No consecutive drama
Cinematic Moment, Signal Interrupt, and Tension Arc cannot follow each other. At least two non-dramatic blocks must separate high-intensity blocks.
🎛️
Dramatic budget per lesson
Max 1 Cinematic Moment, 1 Signal Interrupt, and 1 Tension Arc per lesson. Max 2 high-intensity blocks total. The rest must be calm or building.
🎨
Typography locked at course level
Typography system never changes within a course. Module and lesson variation modulates weight, size, spacing — not the type family itself.
🌡️
Colour volatility budget
Within a module, accent hue can shift ±15° from the course primary. Lesson accents cycle within that range. No jarring hue jumps between consecutive lessons.
🧭
Personality hierarchy
Lesson personality modulates — never overrides — course archetype. A Calm lesson in a Bold archetype course is still bold. A Warm archetype course has no Stark lessons.
📏
Block count range
Minimum 8 blocks per lesson. Maximum 16. The Conductor targets 10–13 as the optimal range. Never fewer — never padded beyond utility.
⏱️
Animation performance budget
Max 3 simultaneously animated elements per viewport. Heavy blocks (Neural Map, Scroll Story) code-split and lazy-loaded. No layout shift after first paint.
🔒
Signature moments are earned
Neural Map only fires once per course (the most concept-dense lesson). Time Capsule only in lesson 1–3. Cinematic Moment only for breakthrough concepts, never routine content.

Implementation Plan

Five phases. Conductor first, always.

The phases are ordered by leverage, not complexity. The Conductor is built first because everything else is decorative without it. Each phase can be tested in production independently.

01
The Conductor — Lesson Rhythm Engine
The intelligence layer. Orchestration logic, anti-chaos rules, signature moment placement, five-beat arc scoring.
Highest leverage
  • Define block emotional weights (calm / building / tension / insight / reward) in a config file
  • Build Conductor function: takes lesson arc position + module mood → returns block sequence constraints
  • Implement dramatic density budget (max 2 high-intensity blocks per lesson)
  • Implement consecutive-drama prevention rule
  • Implement signature moment placement logic (Neural Map once per course, Cinematic Moment gated)
  • Wire Conductor output into LessonExpanderAgent prompt construction
  • Add lesson arc position field to DB (lesson_index / total_lessons / module_position)
02
Module DNA — The Missing Middle
Give every module its own visual district: hue-shifted accent, mood, header style, transition character.
High impact
  • Add module_dna JSONB column to course_topics table (Supabase migration)
  • Build generateModuleDNA(courseId, moduleIndex, courseDNA) — derives hue shift, mood, header style
  • Implement 4 module moods: opening (airy, spacious) / building (denser) / climax (high contrast) / resolution (warm, settled)
  • Build 5 module header component variants
  • Wire ModuleDNA into layout server component and ModuleDNAProvider context
  • Create module entry transition — brief atmospheric shift when navigating between modules
  • Expose module mood as a CSS custom property so block components can read it
03
Semantic Matching + DNA Expansion
Course topic drives archetype bias. Expand palettes to 48, archetypes to 18, typography to 10. Add 6 new visual worlds.
High visual impact
  • Add topic_tags[] to courses table; populate for existing courses
  • Build SEMANTIC_BIAS_MAP: topic tags → archetype weight array (biases, not forces)
  • Expand course-dna-catalogue.ts: 18 archetypes, 48 palettes, 10 typography, 8 bg treatments
  • Add 6 new visual worlds as distinct archetype + palette combinations
  • Implement biased DNA generation: SHA-256 seed modified by semantic weights
  • Re-generate DNA for all existing courses with new catalogue
  • Expand lesson accent cycle from 4 to 12 colours
  • Implement 6 lesson emotional personalities affecting all 10 rotating block components
04
New Block Types — 15 New Instruments
Build all 15 new block components. Cinematic Moment, Neural Map, Signal Interrupt, Time Capsule and 11 more.
Feature richness
  • Priority tier 1 (signature features): Neural Map, Signal Interrupt, Time Capsule, Cinematic Moment
  • Priority tier 2 (interaction): Depth Charge, Contrast Duel, Myth Buster, Perspective Toggle, Tension Arc
  • Priority tier 3 (data/media): Animated Stat, Reality Anchor, Code Cinema, Quote Mosaic, Scroll Story, World Stage
  • Add all 15 to lesson-blocks.ts type definitions and content-sanitizer.ts repair rules
  • Wire Time Capsule cross-lesson persistence to Supabase user_predictions table
  • Expand block layout modes from 3 to 8 for the existing 10 rotating blocks
  • Add micro-variation seeding (border-radius, glow, spacing offsets) to all blocks
05
Performance + Polish
Code-split heavy blocks. Animation budget enforcement. Lazy loading. Luxury UX means zero-friction response.
Mandatory
  • Dynamic import() for heavy blocks: Neural Map, Scroll Story, World Stage, Code Cinema
  • Intersection Observer lazy activation for all animated blocks (no off-screen animation)
  • Animation frame budget: max 3 simultaneously animated elements enforced via Conductor
  • Preload next lesson's block JS during idle time on current lesson
  • CLS elimination audit — all blocks declare min-height before content loads
  • Motion preference respect: prefers-reduced-motion disables non-essential animations globally
